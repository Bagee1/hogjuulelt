Lab Assignment Week 7: Advanced CRUD & Defensive UI
Course: ICSI486 Software Construction | Project: NUM Thesis Management System
Context: Our UI can read and create data. This week, we implement the missing pieces: Update and Delete. We will introduce "Master-Detail" navigation (clicking a table row fills the form) and make our UI "Defensive" by catching validation exceptions from the Core and showing user- friendly popup dialogs.
User Stories (Sprint 7)
• US 1 (Master-Detail): As a user, when I click on a row in the table, the text fields should populate with that record's data, and the application should remember the selected ID.
• US 2 (Defensive Delete): As a user, when I click "Delete", I want to see a confirmation dialog (JOptionPane) before the record is permanently deleted.
• US 3 (Robust Save/Update): As a user, if I leave a field empty or enter invalid data, the application must not crash. Instead, it must catch the IllegalArgumentException from the Core and display a warning popup.
