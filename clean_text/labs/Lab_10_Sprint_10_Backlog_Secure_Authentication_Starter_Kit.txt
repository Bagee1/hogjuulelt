Lab 10 Sprint 10 Backlog: Secure
Authentication
Epic: As a system administrator, I want to secure the web application using Session Management so that only authenticated users can access, create, or modify data.
User Stories & Acceptance Criteria
User Story 1: The Login View & Controller
As a user, I want to see a login page and submit my credentials.
• [ ] A login.jsp is created with a POST form containing username and password inputs.
• [ ] A LoginServlet handles the POST request.
• [ ] If credentials are valid (hardcode admin / password123 for this sprint, or use the DB if ready), the user object is stored in request.getSession().setAttribute("user", username).
• [ ] If invalid, the user is redirected to login.jsp-error=invalid.
User Story 2: The Bouncer (Route Protection)
As a security officer, I want to ensure that no unauthenticated user can access the dashboard.
• [ ] The DashboardServlet checks the session for the "user" attribute before loading any data.
• [ ] If the attribute is null, the servlet immediately issues a response.sendRedirect(request.getContextPath() + "/login") and returns.
• [ ] Only authenticated users can see the dashboard JSP.
User Story 3: The Logout Mechanism
As an authenticated user, I want to securely end my session.
• [ ] A "Logout" link is added to the main navigation (only visible if logged in).
• [ ] A LogoutServlet handles the request by calling session.invalidate().
• [ ] The user is redirected back to the login page after successful logout.

@Override protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException { // Show the login form request.getRequestDispatcher("/WEB- INF/views/login.jsp").forward(request, response); } @Override protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException { String user = request.getParameter("username"); String pass = request.getParameter("password"); // TODO: Replace hardcoded check with your Domain Core (e.g., UserRepository) if ("admin".equals(user) && "password123".equals(pass)) { // Success: Create session and store user HttpSession session = request.getSession(); session.setAttribute("loggedInUser", user); // Post-Redirect-Get pattern response.sendRedirect(request.getContextPath() + "/dashboard"); } else { // Failure: Redirect back with error flag response.sendRedirect(request.getContextPath() + "/login-error=true"); } } }

3. LogoutServlet.java (Session Invalidation)
Java
@WebServlet("/logout") public class LogoutServlet extends HttpServlet {
@Override protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
HttpSession session = request.getSession(false); // false means:
don't create a new one if none exists if (session != null) { session.invalidate(); // Destroy the state on the server } response.sendRedirect(request.getContextPath() + "/login"); } }
