Lab Assignment Week 6: Swing GUI Basics (Read & Create)
Course: ICSI486 Software Construction | Project: NUM Thesis Management System
Context: Over the past weeks, we built a robust Hexagonal Core and connected it to an H2 database. Today, we attach our first Primary Adapter: A Desktop GUI using Java Swing. The UI will remain "dumb" – it contains no business logic and simply communicates with the ThesisService via our Factory.
User Stories (Sprint 6)
• US 1 (Wiring): As a developer, I want to retrieve the ThesisService from the
RepositoryFactory (or ServiceFactory) and pass it to my MainFrame, so the UI can communicate with the Core.
• US 2 (Table Model): As a user, I want to see all database entries in a JTable. The ThesisTableModel must map the Thesis Java Record components (ID, Title, Score/Student) to the correct table columns.
• US 3 (Create Action): As a user, I want to type a title and a score into text fields and click "Save" to persist a new thesis in the database and immediately see it in the table.
