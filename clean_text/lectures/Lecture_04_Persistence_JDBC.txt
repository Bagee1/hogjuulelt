Security Critical
Danger: SQL Injection
THE #1 SECURITY VULNERABILITY IN DATABASES
VULNERABLE CODE (Do NOT use) FATAL ERROR
// Attack: User inputs "' OR '1'='1"
String sql = "SELECT * FROM users WHERE name = '" + userInput + "'";
// Result: SELECT * FROM users WHERE name = '' OR '1'='1'
// Database returns ALL users (Bypasses authentication!)
String Concatenation treats user input as executable SQL code.
SECURE CODE (PreparedStatement) SAFE
// Secure: Use placeholders (-)
String sql = "SELECT * FROM users WHERE name = -";
PreparedStatement stmt = conn.prepareStatement(sql); stmt.setString(1, userInput); // Input treated strictly as data Placeholders separate SQL logic from data. The database is never tricked.

Data Transfer Pattern
What is a DTO-
DTO Pattern Visualization
Data Transfer Object
A DTO is an object that carries data between processes. Its sole purpose is to transfer data, without containing any business logic. It's often a simple POJO or Record.
Why use it-
Decoupling & Security:Decouples the internal domain model from the external API view.
It hides sensitive data (passwords, internal IDs) and sends only what the client needs.
Architecture Layering
The DTO pattern creates a boundary between the Service Layer and the Presentation Layer. It allows the internal model to evolve HERE independently of the external API contract.

Architectural Design
Domain Entities vs. DTOs
Understanding the critical separation between your internal business model and the data you expose to the world is key to a robust Hexagonal Architecture.
Domain Entity Core / Internal Data Transfer Object Boundary / External
Location Inside the Core Domain(Center of Hexagon) At the Boundaries(Adapters / API)
Behavior Rich Logic (Calculations, Validation Rules) None (Just Getters/Setters, "Dumb" Container) State Mutable (State changes over lifecycle) Immutable (Read-only snapshot of data) Structure Complex (Relationships, Hidden Fields) Flat & Tailored (Optimized for UI/Network) Security Contains Sensitive Data (Passwords, Internal IDs) Safe (Sensitive data is stripped out) Pro Tip: Never pass your Entities directly to the UI!
