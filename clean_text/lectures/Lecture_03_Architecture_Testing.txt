Concurrency: The Classic Race Condition
ICSI486 Software Construction
Unsafe Read-Modify-Write
Operations like count++ are not atomic. Threads can read stale values before writing back.
1 2
Common Symptoms
Incorrect counters, lost updates, and intermittent failures that are hard to reproduce.
Thread A Thread B
The Fix: Synchronization
Use atomic types (e.g., AtomicInteger) or locks to ensure mutual Shared Counter exclusion.
Race Condition
Concurrent Access Error
Choose Simplicity
Prefer the simplest correct mechanism. Don't use complex locks if an atomic type suffices.
