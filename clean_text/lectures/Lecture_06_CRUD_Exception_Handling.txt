Graceful Degradation in the UI
Defensive Programming
The UI must catch ALL exceptions at the boundary. Use a layered try-catch strategy to distinguish between user mistakes and system failures.
Two-Layer Catch Strategy
Layer 1: Domain Errors
Specific exceptions (e.g., validation) caused by user actions.
IllegalArgumentException
Layer 2: System Errors
Unexpected failures (DB offline, Network) not caused by the user.
Exception SQLException
