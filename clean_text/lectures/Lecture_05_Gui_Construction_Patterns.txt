The Problem with Absolute Positioning
Fragile Design
Why does it fail-
The Method
Defining exact pixel coordinates and dimensions for every single
Screen Resolutions component.
A 100px wide button looks fine on 1080p, butis microscopic on a 4K monitor. The interface does not scale.
Avoid
JButton btn = new JButton("Save");
// Hardcoded (x, y, width, height) btn.setBounds(100, 200, 80, 30); frame.setLayout(null);
Window Resizing
When users maximize the window, your components stay stuck in the top-left corner, leaving huge empty spaces.
Visual Consequence
Save
OS Font Settings
Overlapping!
If a user sets their OS font size to 150% for accessibility, the text will overflow the fixed button boundaries.
Components clash when fonts change size.
The Solution: Layout Managers
Delegate positioning to intelligent algorithms (BorderLayout, FlowLayout, GridBagLayout).

Dependency Injection

Architecture Rule
The UI Adaptermust NOTinstantiate the Core! It should never know about specific implementations like JdbcThesisRepository. Always program against the interface.
WRONG: Tight Coupling new ThesisService(new JdbcThesisRepository(...))
Hardcoding dependencies inside the UI breaks the architecture. The UI shouldn't decide which database to use.
RIGHT: Factory Pattern
ServiceFactory.getInstance().getThesisService()
Delegates creation logic to the Factory from Week 4. The UI receives a fully configured service instance.
