U Topic 2 The Controller
M Processing POST Requests
Routing Logic ThesisServlet.java
When a form uses method="POST", the Tomcat server inspects the HTTP header. It bypasses doGet() entirely and routes the request to doPost().
Servlet doPost() Method
You must override this specific method in your Servlet class. It provides the HttpServletRequest object containing the body of the submitted form data.
Security & Idempotency
Unlike GET, doPost() is intended for non-idempotent actions—operations that change the server's state (Create, Update, Delete).

Security Alert Web Adapters Part II - CRUD Operations
U
The Danger of GET for Deletion
M
Why is <a href="..."> dangerous-
Search Engine Bots (Crawlers)
Google bots follow every link on a page to index it. If you use GET links for deletion, the bot will delete your entire database!
Browser Pre-fetching
Modern browsers load links in the background to speed up navigation. The browser might trigger the delete URL delete URL automatically without the user clicking.
NEVER use GET! Accidental Sharing
Destructive Actions GET requests are visible in the URL. Users might copy/paste the link to a friend, causing the friend to unintentionally unintentionally delete the record.
THE GOLDEN RULE:
Always use HTTP POST(or DELETE in REST) for data removal.
